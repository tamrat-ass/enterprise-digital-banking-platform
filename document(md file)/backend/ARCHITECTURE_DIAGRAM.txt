BEFORE: N+1 Query Pattern in /api/users
═════════════════════════════════════════

Request: GET /api/users

Step 1: Fetch all users
        ┌─────────────────────────────┐
        │ Query 1: SELECT * FROM user │
        │ Result: 100 users           │
        │ Connections used: 1/5       │
        └─────────────────────────────┘
                    │
                    ▼
For each of 100 users:
        ┌──────────────────────────────────────────┐
        │ Query 2: SELECT * FROM userRoles         │
        │          WHERE userId = user[0].id       │
        ├──────────────────────────────────────────┤
        │ Query 3: SELECT * FROM userRoles         │
        │          WHERE userId = user[1].id       │
        ├──────────────────────────────────────────┤
        │ ...                                      │
        ├──────────────────────────────────────────┤
        │ Query 101: SELECT * FROM userRoles       │
        │            WHERE userId = user[99].id    │
        └──────────────────────────────────────────┘

Total: 1 + 100 = 101 QUERIES ❌
Time: 20+ seconds ❌
Connections held: All 5 ❌


AFTER: Optimized Single JOIN Query
═══════════════════════════════════

Request: GET /api/users

Step 1: Single JOIN query
        ┌────────────────────────────────────────────────┐
        │ SELECT u.id, u.name, u.email,                  │
        │        ur.id, ur.roleId,                       │
        │        r.id, r.name                            │
        │ FROM user u                                    │
        │ LEFT JOIN userRoles ur ON u.id = ur.userId    │
        │ LEFT JOIN roles r ON ur.roleId = r.id         │
        │ ORDER BY u.name                                │
        │ LIMIT 100                                      │
        └────────────────────────────────────────────────┘
                    │
                    ▼
Step 2: Group results in application
        ┌────────────────────────────────────────┐
        │ Result: {                              │
        │   id: user1,                           │
        │   name: "John",                        │
        │   roles: [role1, role2]                │
        │ }                                      │
        │ ...                                    │
        └────────────────────────────────────────┘

Total: 1 QUERY ✓
Time: 50ms ✓
Connections held: 1/20 ✓

Improvement: 100x fewer queries!

  MONITORING QUERIES
═══════════════════════════════════════════════════════════════════════════════

Check current connections:
  SELECT count(*) as total_connections FROM pg_stat_activity;

Check connection breakdown:
  SELECT state, count(*) FROM pg_stat_activity GROUP BY state;

Check for idle transactions (leaks):
  SELECT usename, state, state_change, query
  FROM pg_stat_activity
  WHERE state = 'idle in transaction'
  ORDER BY state_change;

Check query performance:
  SELECT query, mean_time, calls
  FROM pg_stat_statements
  ORDER BY mean_time DESC
  LIMIT 10;
