================================================================================
                       IMPLEMENTATION DETAILS
================================================================================

Sign-In Flow (Now Working):
  1. User submits email/password
  2. Endpoint finds user in database
  3. Validates user is active
  4. Validates email is verified
  5. Finds credential account
  6. Compares password with hash using bcryptjs
  7. Creates session if valid
  8. Returns user data + session info
  9. Sets httpOnly cookie
  10. Frontend redirects to dashboard

Password Storage:
  - Algorithm: bcryptjs
  - Rounds: 12
  - Salt: Auto-generated in hash
  - Format: $2b$12$[salt][hash] (60 characters)
  - Stored in: account.password

Session Management:
  - Token: Cryptographically random (hex)
  - Storage: httpOnly cookie (cannot access via JS)
  - Duration: 7 days
  - Renewed: On each request
  - Secure: Only sent over HTTPS in production
