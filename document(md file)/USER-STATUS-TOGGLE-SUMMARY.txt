================================================================================
                            HOW IT WORKS
================================================================================

User Status Values:
  • active   = User can sign in and access system
  • disabled = User cannot sign in (blocked)
  • invited  = User hasn't completed invitation yet

Status Column Display:
  • Status Badge: Green (active), Yellow (disabled), Blue (invited)
  • Toggle Button: Only for active/disabled users
  • No Button: Invited users cannot be toggled

Toggle Action:
  1. Admin clicks power button in Status column
  2. Dialog confirms action: "Are you sure?"
  3. Button shows loading state
  4. API updates database
  5. Status badge refreshes
  6. Success message displays

================================================================================
                          ADMIN UI USAGE
================================================================================

Step 1: Go to User Management
  → /admin/users or /users

Step 2: Find User to Toggle
  → Use search or role filter

Step 3: Click Power Button
  → In Status column (red PowerOff for active, green Power for disabled)

Step 4: Confirm Action
  → "Are you sure you want to [activate/deactivate] this user?"

Step 5: Status Updates
  → Badge changes immediately
  → Success message appears

Step 6: Impact
  → If deactivated: User cannot sign in anymore
  → If activated: User can sign in again

================================================================================
                          API ENDPOINT
================================================================================

Endpoint: POST /api/users/toggle-status

Request Body:
  {
    "userId": "user_9afe6a178cc20743d123d660",
    "newStatus": "disabled"  // Optional: "active" or "disabled"
  }

Success Response (200):
  {
    "success": true,
    "message": "User status changed to disabled",
    "userId": "user_9afe6a178cc20743d123d660",
    "newStatus": "disabled",
    "previousStatus": "active"
  }

Error Responses:
  400: Missing userId
  400: Invalid status
  404: User not found
  500: Server error
