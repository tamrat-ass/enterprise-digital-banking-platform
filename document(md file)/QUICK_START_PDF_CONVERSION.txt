FAST SETUP (Choose One)
=======================

OPTION 1: USE LIBREOFFICE (Self-Hosted, Free)
----------------------------------------------
Windows:
  1. Download LibreOffice from https://www.libreoffice.org/download/
  2. Run installer, complete installation
  3. In terminal:
     npm install libreoffice-convert
  4. Restart dev server (Ctrl+C, then npm run dev)

macOS:
  1. brew install libreoffice
  2. npm install libreoffice-convert
  3. npm run dev

Linux:
  1. apt-get install libreoffice
  2. npm install libreoffice-convert
  3. npm run dev

OPTION 2: USE CLOUDCONVERT (Cloud, Free Tier Available)
---------------------------------------------------------
1. Go to https://cloudconvert.com and sign up (free)
2. Get API key from settings page
3. Add to .env.local:
   CLOUDCONVERT_API_KEY=your_api_key_here
   CLOUDCONVERT_ENABLED=true
4. Restart dev server (Ctrl+C, then npm run dev)

OPTION 3: NO SETUP NEEDED (Use Now, Fallback Mode)
---------------------------------------------------
System already works! Just test:
1. Go to http://localhost:3000/upload
2. Upload a DOCX file (or any Office file)
3. Go to File Management
4. Click "Preview" - you'll see the original file

TROUBLESHOOTING
===============

"No PDF showing in preview"
  → Check if LibreOffice is installed: libreoffice --version
  → Or check if CloudConvert API key is set in .env.local
  → Or just use it as-is (shows original file, still works)

"PDF conversion not happening"
  → Check server logs for [PDFConversion] messages
  → Verify LibreOffice installed (Windows: run libreoffice.exe)
  → Verify .env.local has CLOUDCONVERT_API_KEY if using CloudConvert
  → Try uploading a smaller file (large files may timeout)

"Upload button not working"
  → Make sure you're logged in
  → Check browser console for errors (F12)
  → Make sure file is under 50MB

"Server won't start"
  → Make sure port 3000 is available
  → Kill existing processes: Get-Process node | Stop-Process
  → Try again: npm run dev
