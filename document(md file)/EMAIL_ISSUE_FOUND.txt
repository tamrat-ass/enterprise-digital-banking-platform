WHY THIS HAPPENS:
──────────────────

  Gmail has TWO password types:

  1. Regular Password
     • Used to sign into Gmail normally
     • Example: tame@4840yene@48
     • ❌ DOESN'T WORK with SMTP

  2. App Password
     • Used for apps/SMTP applications
     • Generated at: myaccount.google.com/apppasswords
     • 16 characters long
     • ✅ WORKS with SMTP


WHAT TO DO (5 MINUTES):
──────────────────────

  Step 1: Go to Gmail Settings
    → URL: https://myaccount.google.com/apppasswords
    → Login with your Gmail if needed

  Step 2: Generate App Password
    → Select: Mail
    → Select: Windows Computer
    → Click: Generate
    → Copy: 16-character password (no spaces)

  Step 3: Update Configuration
    → Open: .env.local
    → Find: SMTP_PASSWORD=PLACEHOLDER_WAITING_FOR_APP_PASSWORD
    → Replace with: Your 16-char app password
    → Save

  Step 4: Restart Server
    → Press: Ctrl+C (stop)
    → Type: npm run dev (start)
    → Wait: "✓ Ready" message

  Step 5: Test
    → Go to: http://localhost:3000/admin/users
    → Create new user (different email)

    → ✅ Should arrive now!
